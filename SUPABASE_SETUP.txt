# Supabase Database Setup Guide

## Step 1: Execute SQL Migration

1. Go to your Supabase project: https://ocutxszgjnmztdawuwgt.supabase.co
2. Navigate to **SQL Editor** → **New Query**
3. Copy the entire content from `sql-migration.sql`
4. Paste it into the query editor
5. Click **Run**

This will create:
- `listings` table with 3 sample Airbnb properties
- `bookings` table for storing reservations
- Necessary indexes for performance

## Step 2: Update Environment Variables

In `src/environments/environment.ts`, verify Supabase credentials are already set:

  url: 'https://ocutxszgjnmztdawuwgt.supabase.co',

1. Go to your Netlify site settings → **Environment variables**
2. Add:
   ```

   SUPABASE_ANON_KEY=sb_publishable_7-jupaDnqhSq7u-vrs5Cpg_4DsoOYgQ

```

Then navigate to `/listings` to see your properties.

## API Endpoints

### Listings
- `GET /listings` - Get all properties
- `GET /listings/:id` - Get property details

### Bookings
- `POST /bookings` - Create new booking
- `GET /bookings?email=user@example.com` - User's bookings

## Sample Data

Three properties are seeded:
1. **Luxe Penthouse Paris** - 185€/night, 3BR, 2BA, 6 guests
2. **Villa Côte d'Azur** - 225€/night, 4BR, 3BA, 8 guests
3. **Appartement Lyon Centre** - 125€/night, 2BR, 1BA, 4 guests

## Row Level Security (RLS) - Optional

For production, enable RLS on Supabase:

1. Go to **Authentication** → **Policies**
2. Enable RLS on `listings` table (public read)
3. Enable RLS on `bookings` table (user can only see their own bookings)

This prevents unauthorized access.

## Notes

- Netlify free tier has no backend, so Supabase handles all data
- Stripe webhook will need a Netlify Function for payment confirmation
- CORS is enabled by default on Supabase for public data
